Typography, colors, sizing

A1

Font Family — Everywhere

Design Decision Needed
Figma uses Sequel Sans. The app uses Sofia Pro (via Theme.semantic.fontFamilies.default.value). Sequel Sans has zero presence in the repo — no .otf/.ttf files, no references anywhere. This is the primary reason the app looks softer and less sharp than the Figma and competitors like WintWealth (which use modern geometric sans-serifs with clean strokes). Needs a product/design decision before it can be fixed: either procure Sequel Sans and add font files to the bundle, or the designer updates Figma to accept Sofia Pro as the shipped font.

Part B — Engineering Gaps

Responsiveness, bugs, best practices

B1

Font not scaling on large phones

P1
Confirmed by engineering. All fontSize values across every homeV3 style file are hardcoded absolute integers. No PixelRatio.getFontScale(), no Dimensions-based scaling, no maxFontSizeMultiplier on any <Text>. On a 430pt iPhone Pro Max vs a 375pt iPhone SE, the same 14px occupies proportionally less screen real estate — content reads smaller on bigger phones.
// Current — every style file in homeV3 (same hardcoded pattern) issuerName: { fontSize: 14 }, // identical on 375pt SE and 430pt Pro Max ytm: { fontSize: 20 }, title: { fontSize: 16 }, // Fix option — scale relative to a 375pt baseline import { Dimensions } from 'react-native'; const scale = Dimensions.get('window').width / 375; const fs = (size: number) => Math.round(size * scale); // Or cap system font scaling on layout-sensitive text <Text maxFontSizeMultiplier={1.2}>11.75%</Text>

Three approaches can be combined: (1) baseline scale utility for sizes, (2) maxFontSizeMultiplier to cap accessibility scaling on tightly-fitted elements, (3) allowFontScaling={false} only on price/number displays where layout integrity is critical.

B3

All data is mock — zero live API connections

P0 Blocker
Most critical gap. The entire HomepageContainer.tsx imports exclusively from ./mockData/bonds, ./mockData/faqs, ./mockData/goals, and ./mockData/stats. Not a single widget fetches from a live API, Strapi, or user auth state. The UI shell is built but the data layer is entirely absent.
// HomepageContainer.tsx — all data hardcoded import { HIGH_YIELD_BONDS, POPULAR_BONDS } from './mockData/bonds'; import { FAQ_ITEMS } from './mockData/faqs'; import { GOAL_CARDS } from './mockData/goals'; import { POPULAR_PICK_PILLS } from './mockData/stats'; // None of these are wired up: // - Live bond inventory API // - User KYC state (4 states drive CTA + heading) // - Strapi CMS (FAQs, Why Invest cards) // - Popular bonds endpoint (GET /v1/bonds/popular) // - User portfolio (for "Invested ✓" badges)
