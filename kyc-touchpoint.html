Key Insights

What the data says — and what to fix.

🔴 P0 — KycStatusCard has zero analytics (69.7% dark)

The homepage widget for KYC_NOT_STARTED users navigates to KYC_FORM with no event. Once instrumented, the "Other/Uninstrumented" bucket is expected to drop to <20% and this will become the single largest attributed touchpoint.

🟡 P1 — Bond Detail CTA misses KYC status (14.4% last-touch)

BOND_DETAILS_INVEST_NOW_BUTTON fires for all users regardless of KYC state. userCategory (KYC_NOT_STARTED / KYC_PENDING) is not logged on this event. Fix: add userCategory to the event payload so Amplitude can filter.

🔵 Bond browsing drives more KYC than direct CTAs

Section D shows 25+ users reached KYC through bond-browsing paths (High Yield, Highly Rated, Issuer widget). These users hit the KYC wall after discovering a bond — intent signal is strong here. Friction in this path directly kills conversions.

🟡 WAYLF tab disappears in last-touch (0% vs 17% any-touch)

234 users clicked the WAYLF "Bonds" tab before KYC — but in last-touch, they're all reclassified to Bond Detail or Explore. WAYLF is an upstream step, not the final trigger. The tab works for discovery; the conversion point is bond detail.

🟢 Android dominates (89.3%) — optimise for it first

KYC instrumentation gaps (KycStatusCard, NonInvestedHoldingCard) affect all platforms, but fixing them will move Android metrics most. iOS 4.7% is relatively small. Web 2.6% is web-banner driven.

🔵 Portfolio page + Bond Basket have no KYC events either

NonInvestedHoldingCard (portfolio) and BondBasketDetail navigate to KYC_FORM with no analytics. These contribute to the 69.7% dark bucket alongside KycStatusCard.

Methodology — Last-Touch

  • Downloaded 136.8 MB 7-day raw event export from Amplitude Export API.
  • Filtered in-memory to touchpoint events + KYC_RISKDETAILS_SUBMIT only.
  • For each user who fired KYC_RISKDETAILS_SUBMIT in the window: walk backwards through their sorted event stream, find the last instrumented touchpoint event before that timestamp.
  • 1,336 of 1,360 attributed (gap of 24 = users with server-side receipt time outside window).
  • Each user counted exactly once — no double-counting between surfaces.

Note on Overlap — Section A vs B

  • Section A and B are NOT additive. They answer different questions about the same set of users.
  • Section A: "Which CTA button directly triggered the KYC flow?" → credits Bond Detail at 16.8%.
  • Section B: "What brought users to Bond Detail in the first place?" → upstream discovery path.
  • A user who went Homepage High Yield → Bond Detail → Invest Now appears in BOTH sections.
  • Section D (last-touch above) resolves this by crediting each user to exactly one surface.
