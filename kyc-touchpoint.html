BOND_DETAILS_INVEST_NOW_BUTTON fires for all user types. userCategory (KYC_NOT_STARTED / KYC_PENDING) is not logged. Fix: add userCategory to the payload in useInvestNowLogic.ts and useBondDetailContainer.ts.

WAYLF 'Bonds' tab is a discovery path, not a KYC trigger (236 any-touch vs 0 as entry in last-touch)

In any-touch funnels, 236 users clicked the WAYLF Bonds tab before KYC. In 3-hop last-touch, WAYLF shows up as entry path for 0 users — they continued to Bond Detail where the actual trigger happened. Optimise the Explore page UX to convert these browsing users faster.

264 users viewed the Explore tab just before their KYC trigger. These are high-intent users browsing bonds — reducing friction from Explore → Bond Detail → KYC would directly improve conversion.

73% of attributed starts have no tracked entry path

All 4 instrumentation gap fixes (KycStatusCard, NonInvestedHoldingCard, BondBasketDetail, userCategory on Bond Detail) will move Android metrics most. iOS 4.7% and Web 2.6%.

Methodology — 3-Hop Last-Touch

  • Downloaded 7-day raw event export from GET /api/2/export.
  • Filtered in-memory to CTA events + entry events + KYC_RISKDETAILS_SUBMIT.
  • Hop 1: find last CTA event before KYC_RISKDETAILS_SUBMIT → "trigger".
  • Hop 2: find last entry/discovery event before that CTA → "entry path".
  • Each user counted exactly once. No double-counting between surfaces.
  • Comparison to any-touch (Sections A/B below): any-touch counts a user for every surface in the 7d window — 3-hop counts them once for their final path.
