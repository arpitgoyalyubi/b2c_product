Segments use 365-day rolling window — a user who invested 6 months ago and fails PIN today is still counted as "existing investor". PIN failures = hard block (user cannot proceed). PIN_TOKEN_EXPIRED above = soft interrupt (user must re-authenticate, but can continue).

PIN_TOKEN_EXPIRED fires when a user's PIN session token expires mid-session (HTTP 403 from backend). The user is redirected to VERIFY_SECURE_PIN_POST_LOGIN to re-enter their PIN. They are NOT locked out — but any in-flight action (e.g. mid-checkout) is interrupted. If the "Existing investors" row spikes, it indicates active payment-ready users hit a session wall → correlates with payment drop.
