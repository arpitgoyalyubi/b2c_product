Column Glossary
LV Users*
MOBILE_LAUNCH_VIEW unique users — proxy for installs. AppsFlyer install data excluded due to iOS ATT attribution gaps and reinstall inflation. Treat as "users who opened the pre-signup launch screen" rather than fresh installs.
Views/User
metric=totals ÷ metric=uniques — avg times MOBILE_LAUNCH_VIEW fires per unique user over the period. >2× indicates many users return to the launch screen before signing up.
Launch CTA
MOBILE_LAUNCH_CTA_CLICKED — user taps the CTA button on the pre-signup launch screen. Drop here = users seeing the screen but not engaging with the CTA.
Signin PV
SIGNIN_PAGE_VIEW · new users only (new_active segment — same as Iris Pulse Stage 1)
OTP Sent
SIGNIN_PAGE_VERIFY_API_SUCCESS — phone number submitted & OTP dispatched by backend
OTP Screen
VERIFY_OTP_PAGE_VIEW — user reached the OTP entry screen
OTP Entered
VERIFY_OTP_ENTERED — user typed an OTP code
OTP ✓
VERIFY_OTP_SUCCESS — OTP verified (Mobile OTP ✓ — Iris Pulse Stage 1 key outcome)
Email PV
EMAIL_PAGE_VIEW — Stage 2 entry · fetched with same new_active segment as Stage 1 funnel so OTP✓→Email% is consistent with the Stage 1 population
SSO path %
SSO_VERIFICATION_SUCCESS ÷ OTP ✓ — share of new users who verified via Google Sign-In. Google SSO launched Mar 6 2026. SSO users skip EMAIL_PAGE_VIEW entirely — rising SSO adoption will reduce Email PV counts even if total registrations are stable or growing.
Signup
SETUP_SECURE_PIN_SUCCESS — registration complete
LV*→Signup
Signup ÷ LV Users* — end-to-end proxy rate (Web: Signin PV → Signup)
WoW
LV*→Signup current 30d vs previous 30d in percentage points
Colour coding
Each step shows conv% from the prior step · green = on target · amber = watch · red = investigate
CTA% Drop · Root Cause Hypothesis Ranking

Repeat-user CTA% = 98% (logged-out users landing on same screen) — rules out technical glitch and UI bugs as primary drivers. Ad group CTA% gap: In-market audience 92% vs Bonds_UACe 37% on the same screen — confirms user intent quality as primary lever.

Hypothesis Strength Key Evidence / Reasoning
User intent quality degradation from spend mix Very Strong Ad group data shows 2.5× CTA% gap on same screen. As budget shifts to broad targeting (opentargeting, UACe), cohort intent degrades.
Ad creative ↔ landing screen disconnect Strong UAC auto-creatives may show generic investment imagery; screen says "bonds" specifically. Expectation mismatch causes drop-off for broad-match users.
“Bonds” category awareness gap Strong Repeat users (98% CTA%) know bonds. New broad-acquisition cohorts have zero mental model. Screen doesn’t answer “what is this?” before asking them to tap.
Missing social proof (first-time users) Moderate No “1L+ investors / ₹500cr invested” signal. Repeat users don’t need it. New low-intent users need external validation before committing to tap.
Cold start render latency (Android Low) Moderate Largest cohort (~23k LV). Warm app = fast render = 98% repeat CTA%. Cold start on low-RAM devices may delay CTA button rendering; user taps blank space and exits.
Language / comprehension barrier Moderate Spend expansion into tier-2/3 cities likely brings non-English users. Screen is English-only. “Fixed returns”, “SEBI Registered Broker” require financial literacy in English.
Carousel confusion (2-slide ambiguity) Weak–Moderate Slide 1 (returns) → Slide 2 (security) sends two signals. Users may wait for more slides. Hard to isolate without A/B test.
Visual / UI issues (button, SEBI badge) Weak Real imperfections but repeat users (same screen) show 98% CTA%. Marginal effect at most; not a primary driver.
Technical glitch Ruled out 98% repeat-user CTA% on the same screen closes this completely.

Highest-leverage fix: Shift budget back toward in-market / purchase-signal targeting (already 92%+ CTA%). Parallel track: add social proof (“1L+ investors”) + one-line bond explainer (“bonds = fixed deposits with higher returns, SEBI-regulated”) to the pre-signup screen to convert lower-intent new cohorts.
