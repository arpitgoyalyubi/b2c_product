Column Glossary
Install
[AppsFlyer] Install — fires once on fresh app install (mobile only; web has no install event)
Launch View
MOBILE_LAUNCH_VIEW — pre-signup launch screen comes into focus (mobile only, fires before sign-in). All-user count (no new-user filter needed — fires only pre-signup).
Launch CTA
MOBILE_LAUNCH_CTA_CLICKED — user taps the CTA button on the pre-signup launch screen. Drop here = users seeing the screen but not engaging with the CTA.
Signin PV
SIGNIN_PAGE_VIEW · new users only (new_active segment — same as Iris Pulse Stage 1)
OTP Sent
SIGNIN_PAGE_VERIFY_API_SUCCESS — phone number submitted & OTP dispatched by backend
OTP Screen
VERIFY_OTP_PAGE_VIEW — user reached the OTP entry screen
OTP Entered
VERIFY_OTP_ENTERED — user typed an OTP code
OTP ✓
VERIFY_OTP_SUCCESS — OTP verified (Mobile OTP ✓ — Iris Pulse Stage 1 key outcome)
Email PV
EMAIL_PAGE_VIEW — Stage 2 entry · fetched with same new_active segment as Stage 1 funnel so OTP✓→Email% is consistent with the Stage 1 population
SSO path %
SSO_VERIFICATION_SUCCESS ÷ OTP ✓ — share of new users who verified via Google Sign-In. Google SSO launched Mar 6 2026. SSO users skip EMAIL_PAGE_VIEW entirely — rising SSO adoption will reduce Email PV counts even if total registrations are stable or growing.
Signup
SETUP_SECURE_PIN_SUCCESS — registration complete
I→Signup
Signup ÷ Install — end-to-end install→registration rate (Web: Signin PV → Signup)
WoW
I→Signup current 30d vs previous 30d in percentage points
Colour coding
Each step shows conv% from the prior step · green = on target · amber = watch · red = investigate
